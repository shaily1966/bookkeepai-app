- Add "FEE_REBATE_PAIR" to Notes on both rows.

POLARITY RULES (CRITICAL)
━━━━━━━━━━━━━━━━━━━━━━━━━━
Credits (negative amounts) must be classified correctly:
- PAYMENT (e-transfer, bank payment) → Type = PAYMENT. NOT revenue.
- REFUND (merchant return, credit from store) → Type = REFUND. This is an expense reduction.
- FEE_REBATE (fee waiver, annual fee rebate) → Type = FEE_REBATE. This is a fee offset.

NEVER classify merchant refunds as revenue/income.
NEVER classify fee rebates as revenue/income.
Only classify credits as income if they are explicitly client payments, sales deposits, or e-transfer receipts for services.

PROVINCIAL TAX / ITC RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━
Default province from context header. Apply correct tax rate:

Ontario (ON): ITC = Debit × 13 / 113 (HST 13%)
Nova Scotia / New Brunswick / Newfoundland / PEI: ITC = Debit × 15 / 115 (HST 15%)
Alberta (AB): ITC = Debit × 5 / 105 (GST only, no PST)
British Columbia (BC): ITC = Debit × 5 / 105 (GST only; PST not recoverable)
Saskatchewan (SK): ITC = Debit × 5 / 105 (GST only; PST not recoverable)
Manitoba (MB): ITC = Debit × 5 / 105 (GST only; RST not recoverable)
Quebec (QC): ITC = Debit × 5 / 105 (GST only; QST claimed separately)

PROVINCE DETECTION FROM DESCRIPTION:
If description contains a province code at end (e.g. "STORE NAME BC", "MERCHANT QC"):
Apply that province's rate for THAT transaction, even if default province differs.
Add "ITC_PROV=XX" to Notes.

Round ITC to 2 decimals.
If Category = Meals & Entertainment: still calculate ITC, add "MEALS_50_RULE" to Notes.
If Type = PAYMENT or FEE_REBATE: leave ITC blank.
If Category = Owner Draw / Personal: leave ITC blank.
If Category = Insurance: leave ITC blank (exempt supply).
If Category = Bank Charges: leave ITC blank (financial service, exempt).

HOW TO IDENTIFY REAL TRANSACTION ROWS (extract these):
- Has a single specific date (not a date range)
- Has a merchant name, payee name, or bank transaction code
- Has a specific dollar amount that is NOT equal to the sum of nearby rows
- Appears in the transaction body area, not in a header or footer

HOW TO IDENTIFY SUMMARY LINES (skip these):
- Contains a date range ("Apr 17 to May 16") not a single date
- Is a section title (e.g. "Payments received...", "New charges", "Purchases")
- The amount equals the sum of other rows in the same section
- Appears before the individual transaction rows of a section

POSTING DATE vs TRANSACTION DATE:
When a statement shows TWO dates per row (Transaction Date and Posting Date),
use the POSTING DATE as the Date column value.
The posting date is the authoritative cycle date used for statement reconciliation.
Example: Transaction Date Jan 28, Posting Date Jan 29 → output "Jan 29" as Date.

CHEQUING ACCOUNT POLARITY:
For chequing/savings accounts, withdrawals are DEBIT (positive), deposits are CREDIT (negative in our system).
- "DR" flag or "-" suffix = withdrawal = DEBIT
- "CR" flag = deposit/incoming = CREDIT
- Positive balance change from last balance = deposit = CREDIT
- Negative balance change from last balance = withdrawal = DEBIT

PAYMENT TYPE IDENTIFICATION (applies to all bank types):
- E-transfer sent / Interac sent → PURCHASE (if to a vendor) or PAYMENT (if repaying a loan/card)
- E-transfer received / Interac received → INCOME (if from a client) or PAYMENT (if from personal)
- "TELESCOTIA PMT" / "CIBC BANK PMT" / "TD PMT" / "RBC PMT" / "BMO PMT" → PAYMENT (credit card payment)
- "PAYMENT - THANK YOU" / "PAYMENT RECEIVED" / "PMT/PAIEMENT" → PAYMENT
- NSF (Non-Sufficient Funds) fee → FEE
- "Overlimit fee" / "Late payment fee" → FEE
